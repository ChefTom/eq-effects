# Three-Pane Layout

## Overview

The Three-Pane Layout organizes interaction into three synchronized views:

1. Conceptual Conversation
2. Structured Mathematical Form
3. Contextual State

---

## Left Pane: Conceptual Conversation

### Purpose
To capture intent and reasoning.

### Functions
- describe problems in natural language
- refine objectives
- express constraints and expectations

---

## Center Pane: Structured Mathematical Form

### Purpose
To define formal mathematical structure.

### Functions
- construct equations
- define variables
- specify relationships and conditions

---

## Right Pane: Contextual State (Contextual Box)

### Purpose
To display and manage system state.

### Functions
- track variables and definitions
- display assumptions and constraints
- indicate execution readiness
- highlight inconsistencies

---

## Synchronization

All panes are synchronized:

- conceptual changes update form and context
- mathematical changes update context and interpretation
- contextual updates influence both input paths

---

## Interaction Model

- users can begin in any pane
- information flows across panes automatically
- the system maintains consistency

---

## Outcome

The Three-Pane Layout provides a **unified interaction model** that integrates reasoning, structure, and state.
