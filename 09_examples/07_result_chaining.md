# Result Chaining

## Overview

Result Chaining defines how outputs from one computation can be used as inputs for subsequent computations.

This enables iterative modeling, refinement, and composition of complex systems.

---

## Purpose

To allow users to:

- build on previous results
- refine models incrementally
- construct multi-stage computational workflows

---

## Model

In EQ Effects:

- each result is a structured artifact (Rich Result Package)
- outputs may be referenced as inputs to new Mathematical Objects

---

## Behavior

The system allows:

- selection of prior results
- extraction of relevant components (values, equations, context)
- integration into new problem definitions

---

## Characteristics

- structured
- traceable
- reproducible

---

## Traceability

Chained results maintain:

- linkage to original computations
- full audit trail across stages
- reproducible execution paths

---

## Example Flow

1. Solve initial model
2. Extract output variables
3. Define new constraints or scenarios
4. Re-execute with updated context

---

## Outcome

Result Chaining enables:

- iterative refinement
- composable workflows
- complex system modeling
