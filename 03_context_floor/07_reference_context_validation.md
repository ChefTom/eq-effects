# Reference Context Validation

## Overview

Reference Context Validation defines how EQ Effects evaluates user work against supporting materials such as papers, notes, scans, or documents.

The goal is not to blindly accept a reference as correct.

The goal is to compare, contextualize, and identify alignment or divergence.

---

## Purpose

To support review-style workflows where a user asks:

- Does my method align with this paper?
- Are my assumptions consistent with this source?
- Did I miss a constraint?
- Is this proposed use novel or already represented in the literature?
- Where does my model diverge from the reference?

---

## Validation Types

### Assumption Comparison
Compares assumptions in the user’s Mathematical Object against assumptions found in reference material.

### Equation Comparison
Identifies whether equations, structures, or transformations align with referenced material.

### Method Comparison
Compares the user’s selected or proposed solution method against documented methods.

### Constraint Comparison
Checks boundary conditions, units, domains, and limitations.

### Novelty-Oriented Review
Identifies whether the user’s proposed application appears different from the reference context.

---

## System Behavior

The system may produce:

- aligned elements
- divergent elements
- missing assumptions
- unsupported claims
- unclear or low-confidence comparisons

---

## Important Boundary

Reference Context Validation is not a substitute for academic peer review.

It is a structured review aid that helps users inspect mathematical alignment, evidence, and reasoning.

---

## Outcome

Reference Context Validation helps users understand how their work relates to existing material while preserving transparency about confidence and limitations.
