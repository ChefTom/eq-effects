# Translation and Accessibility

## Overview

EQ Effects is designed to remove barriers between mathematical reasoning and computational execution.

It supports translation across:

- input modalities
- levels of abstraction
- domain-specific language

---

## Translation Model

The system translates between:

- conceptual descriptions
- formal mathematical expressions
- structured internal representations

This translation is internal and governed.

Users are not required to perform it manually.

---

## Accessibility Goals

- allow domain experts to work without programming knowledge
- support multiple levels of mathematical sophistication
- enable incremental refinement of problems

---

## Input Diversity

The system accepts:

- incomplete inputs
- mixed abstraction levels
- domain-specific terminology

---

## System Responsibility

The system ensures:

- consistent interpretation
- structured representation
- alignment with execution requirements

---

## Outcome

Translation and accessibility mechanisms ensure that:

- users can work in their native reasoning model
- the system adapts to the user
- mathematical intent is preserved
