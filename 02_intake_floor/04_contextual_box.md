# Contextual Box

## Overview

The Contextual Box is a live semantic state container that maintains the current state of the problem.

It integrates information from all input paths.

---

## Contents

The Contextual Box tracks:

- variables and definitions
- units and dimensions
- assumptions
- constraints
- domain classification
- precision expectations

---

## Purpose

To ensure consistency across:

- conceptual input
- mathematical structure
- execution preparation

---

## Behavior

The Contextual Box:

- updates dynamically as input changes
- validates relationships between elements
- identifies missing or conflicting information
- supports system prompts for clarification

---

## Role in the System

It acts as a bridge between:

- user reasoning
- mathematical representation
- execution logic

---

## Characteristics

- persistent across interactions
- structured and inspectable
- synchronized with all input paths

---

## Outcome

The Contextual Box ensures that:

- all relevant information is captured
- inconsistencies are resolved early
- the system maintains a coherent state
