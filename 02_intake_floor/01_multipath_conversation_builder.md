# Multipath Conversation Builder

## Overview

The Multipath Conversation Builder is the primary intake mechanism in EQ Effects.

It allows users to express mathematical work through multiple synchronized input paths.

---

## Input Paths

The system supports three coordinated paths:

1. **Conceptual Conversation**
2. **Structured Mathematical Input**
3. **Contextual State (Contextual Box)**

Each path captures a different aspect of intent.

---

## Design Principle

Users are not required to choose a single mode of interaction.

Instead, they can:

- describe intent conceptually
- define equations formally
- specify assumptions and constraints contextually

All inputs are unified into a single representation.

---

## Synchronization

Changes in one path propagate to others:

- conceptual descriptions inform equation structure
- equations update contextual variables
- contextual constraints refine both conversation and form

---

## Interaction Model

- Users may begin with any path
- The system identifies missing information
- Clarification is requested when needed
- The representation evolves incrementally

---

## Outcome

The Multipath Conversation Builder produces a **coherent intent model** that:

- preserves meaning
- aligns with mathematical structure
- prepares for execution without requiring code
