# Reference Material Intake

## Overview

Reference Material Intake allows users to provide supporting material that informs, validates, or challenges a mathematical workflow.

This material may include:

- scanned notes
- old printouts
- PDFs
- journal articles
- textbook excerpts
- images
- documents
- plain text

Reference material is not treated as the authoritative problem definition.

It is supporting context.

---

## Purpose

To allow users to compare their mathematical work against external references without forcing them to manually translate those references into equations or code.

---

## Supported Use Cases

Reference Material Intake may support:

- comparing a proposed method against a published paper
- extracting assumptions from a technical document
- checking whether a student’s approach aligns with known methods
- identifying gaps between a working model and a reference source
- using historical or legacy material to reconstruct a method

---

## Input Boundary

Reference material may be used to support interpretation, comparison, and validation.

It should not be used to import large proprietary datasets or expose confidential work without authorization.

---

## System Behavior

The system may:

- identify relevant equations
- extract assumptions and constraints
- summarize mathematical claims
- compare methods
- identify differences between user work and reference material
- flag uncertainty or incomplete extraction

---

## Relationship to Intake

Reference Material Intake supports the Multipath Conversation Builder.

It does not replace:

- conceptual conversation
- structured formula input
- contextual state

Instead, it enriches those paths with external mathematical context.

---

## Outcome

Reference Material Intake allows EQ Effects to incorporate supporting evidence into the mathematical workflow while preserving the user’s own problem definition.
