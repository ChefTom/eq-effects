# Domain Style Selection

## Overview

Domain Style Selection allows users to guide how problems are structured and interpreted within a specific domain.

---

## Purpose

To provide structured options without requiring domain-specific programming.

---

## Quantum Example

For quantum systems, users may select:

- gate-based (circuit model)
- Hamiltonian-based (continuous systems)
- probabilistic measurement model
- hybrid classical–quantum

---

## Behavior

The system uses this selection to:

- guide equation construction
- influence variable types
- inform method and execution choices

---

## Characteristics

- optional
- non-restrictive
- integrated with existing input paths

---

## Outcome

Users can guide system behavior without leaving the unified input model.
