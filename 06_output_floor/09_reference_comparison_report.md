# Reference Comparison Report

## Overview

A Reference Comparison Report is an output artifact that compares a user’s mathematical work against provided reference material.

It is generated from:

- the user’s Mathematical Object
- contextual state
- extracted reference material
- validation results

---

## Purpose

To provide a structured report showing how the user’s work relates to external mathematical or scientific sources.

---

## Report Sections

A Reference Comparison Report may include:

### 1. Reference Summary
A concise description of the provided material.

### 2. Extracted Mathematical Claims
Equations, assumptions, methods, constraints, or findings identified in the reference.

### 3. User Work Summary
A structured summary of the user’s Mathematical Object and intended use.

### 4. Alignment
Elements that appear consistent between the user’s work and the reference.

### 5. Divergence
Elements that differ, conflict, or require explanation.

### 6. Missing Context
Assumptions, variables, units, or constraints present in one source but missing from the other.

### 7. Novelty Notes
Areas where the user’s proposed use appears to differ from the reference material.

### 8. Confidence and Limitations
A clear statement of extraction quality, uncertainty, and review limits.

---

## Use in Education

This report can support classroom workflows where students:

- study a known method
- propose new applications
- compare their approach against published work
- identify assumptions and limitations

---

## Boundary

The report is not a claim of correctness or publication-level peer review.

It is a structured comparison tool.

---

## Outcome

The Reference Comparison Report turns external material into usable mathematical context while preserving transparency, uncertainty, and traceability.
