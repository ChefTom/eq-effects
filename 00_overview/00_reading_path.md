# Reading Path

EQ Effects is organized as a layered system (“floors”).  
Each floor builds on the previous to move from **intent → form → execution → result**.

This repository is documentation-first. It is intended to be read in sequence.

---

## Recommended Order

1. [`01_foundation_floor/00_foundation_floor.md`](foundation floor)
   - Why the system exists
   - Core thesis and operating model

2. **Intake Floor**
   - Multipath Conversation Builder
   - Conceptual input, structured math input, contextual state

3. **Context Floor**
   - How meaning is captured
   - Assumptions, units, constraints, and ambiguity resolution

4. **Form Floor**
   - Mathematical Object
   - Equation Graph, Variable Registry, Boundary Conditions

5. **Execution Floor**
   - Computer as Solver
   - Method and library selection
   - Compute mapping and transparency

6. **Output Floor**
   - Rich Result Package
   - Process trace, validation, replayability

7. **Interface Floor**
   - Three-pane interaction model
   - User-facing workflow

8. **Governance Floor**
   - Identity, audit, precision governance
   - Reproducibility and non-proprietary boundaries

9. **Examples**
   - End-to-end walkthroughs demonstrating system behavior

---

## How to Read

- Read linearly for a full system understanding
- Reference individual floors for specific topics
- Use the Glossary for consistent terminology

---

## Outcome

By the end of this reading path, the reader should understand:

- how EQ Effects captures mathematical intent
- how that intent becomes a structured object
- how the computer performs execution
- how results are returned with full transparency
