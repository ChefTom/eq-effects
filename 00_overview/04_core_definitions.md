# Core Definitions

This section defines the foundational terms used throughout EQ Effects.

These definitions are consistent across all floors and documents.

---

## EQ Effects

A governed mathematical execution environment where user intent is transformed into structured mathematical objects and executed by the computer with full audit and transparency.

---

## Multipath Conversation Builder

A flexible intake system that allows users to express mathematical intent through:
- conceptual language
- structured formulas
- contextual inputs

These inputs are unified into a single representation.

---

## Contextual Box

A live semantic state container that tracks:
- variables
- assumptions
- units and dimensions
- constraints
- execution readiness

It bridges conceptual input and formal mathematics.

---

## Contextual Processor

A system component that interprets user input and constructs:
- intent models
- domain context
- structured representations

It ensures consistency between language, math, and execution.

---

## Mathematical Object

The canonical representation of mathematical intent.

It includes:
- equations and relationships
- variables and units
- constraints and assumptions
- precision requirements

It is the primary input to execution.

---

## Equation Graph

A structured representation of relationships between mathematical expressions and variables.

It defines dependencies and evaluation order.

---

## Variable Registry

A structured record of all variables, including:
- definitions
- units
- domains
- constraints

---

## Rich Result Package

A structured output containing:
- final result
- process trace
- assumptions and constraints
- validation and warnings
- replayable execution context

---

## Computer as Solver

A system model where the responsibility for solving is performed by the computer under governed execution, rather than by user-defined software.

---

## Governance Layer

The set of rules and structures that ensure:
- consistency
- transparency
- auditability
- reproducibility

---

## Replayability

The ability to reproduce results using the same mathematical object and execution conditions.

---

## Precision Governance

The management of numerical accuracy and fidelity across different regions of computation.

---

## LIME-Compatible Orchestration

A structured approach to contextual processing where:
- input paths are separated and curated
- reasoning is organized into defined stages
- outputs are synthesized into a coherent result

This is referenced for compatibility, not as a dependency.
