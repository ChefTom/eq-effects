# What EQ Effects Is Not

EQ Effects could be misunderstood when compared to existing tools.  
This section defines what it is not, to prevent incorrect assumptions.

---

## Not a Math Solver

EQ Effects does not position itself as a solver.

It does not:
- implement a single solving engine
- restrict execution to a specific method

Instead, it governs how solving is performed by the computer.

---

## Not a Notebook Environment

EQ Effects is not a replacement for notebook systems.

It does not:
- rely on sequential code cells
- require scripting to express intent
- depend on manual execution flow

---

## Not a Programming Framework

EQ Effects does not require:
- Python, MATLAB, or other scripting languages
- solver-specific APIs
- manual method selection in code

Users are not required to translate intent into programming constructs.

---

## Not an AI Agent System

EQ Effects is not an autonomous agent platform.

It does not:
- operate independently of user-defined intent
- generate solutions without structured input
- rely on opaque decision-making

All outputs are governed, structured, and traceable.

---

## Not a Black Box

EQ Effects does not hide execution.

It does not:
- obscure method selection
- suppress intermediate steps
- return unexplained results

All outputs include process transparency.

---

## Summary

EQ Effects avoids the limitations of existing categories:

| Category             | Limitation                        | EQ Effects Approach               |
|---------------------|----------------------------------|----------------------------------|
| Solvers             | Fixed methods                    | Method-agnostic governance       |
| Notebooks           | Code-first workflows             | Intent-first workflows           |
| Frameworks          | Requires programming             | No forced coding                 |
| AI Systems          | Opaque reasoning                 | Transparent, structured process  |

---

## Outcome

EQ Effects establishes a new category:

> governed mathematical execution

distinct from tools that require users to adapt to software constraints.
