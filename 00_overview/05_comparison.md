# Comparison

## Overview

EQ Effects introduces a new model for mathematical computation.

This section compares it to traditional approaches.

---

## Comparison Table

| Feature | Traditional Systems | EQ Effects |
|--------|-------------------|------------|
| Input Model | Code-based | Concept + Math + Context |
| Workflow | Linear (scripts/notebooks) | Multi-path, synchronized |
| Execution | User-managed | System-governed |
| Output | Values | Structured result package |
| Transparency | Limited | Full process visibility |
| Reproducibility | Inconsistent | Built-in |
| Abstraction | High (code translation) | Low (intent preservation) |

---

## Key Differences

### Input

Traditional systems require translation into programming languages.

EQ Effects captures intent directly.

---

### Execution

Traditional systems require manual control of execution.

EQ Effects governs execution automatically.

---

### Output

Traditional systems return results.

EQ Effects returns structured, reproducible artifacts.

---

## Outcome

EQ Effects represents a shift from:

> tool-driven computation

to:

> system-governed, intent-driven execution
